Detailed Description

Background worker that can be aborted.

WARNING: Calling Abort() method disposes the background worker. Therefore, you should define the RunWorkerAborted event that re-creates the background worker and sets the event handlers on it.

◆ Abort()

void IG.Forms.AbortableBackgroundWorker.Abort ( )
inline

Aborts the current job, if a job is currently perrforming. Otherwise, nothing is done.

The RunWorkerAborted event is raised, with event arguments sett to the object that was passed to the DoWork

This method disposes the current background worker. Therefore, RunWorkerAborted should be defined that re-creates tha background worker, sets all the event handlers on it appropriately, and assigns the newly created background worker to the appropriate variable that was before holding the old one.
