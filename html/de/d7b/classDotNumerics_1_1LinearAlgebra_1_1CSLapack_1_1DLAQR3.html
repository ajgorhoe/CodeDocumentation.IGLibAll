Parameters
WANTT(input) LOGICAL If .TRUE., then the Hessenberg matrix H is fully updated so that the quasi-triangular Schur factor may be computed (in cooperation with the calling subroutine). If .FALSE., then only enough of H is updated to preserve the eigenvalues. /param>
Parameters
WANTZ(input) LOGICAL If .TRUE., then the orthogonal matrix Z is updated so so that the orthogonal Schur factor may be computed (in cooperation with the calling subroutine). If .FALSE., then Z is not referenced. /param>
Parameters
N(input) INTEGER The order of the matrix H and (if WANTZ is .TRUE.) the order of the orthogonal matrix Z. /param>
Parameters
KTOP(input) INTEGER It is assumed that either KTOP = 1 or H(KTOP,KTOP-1)=0. KBOT and KTOP together determine an isolated block along the diagonal of the Hessenberg matrix. /param>
Parameters
KBOT(input) INTEGER It is assumed without a check that either KBOT = N or H(KBOT+1,KBOT)=0. KBOT and KTOP together determine an isolated block along the diagonal of the Hessenberg matrix. /param>
Parameters
NW(input) INTEGER Deflation window size. 1 .LE. NW .LE. (KBOT-KTOP+1). /param>
Parameters
H(input/output) DOUBLE PRECISION array, dimension (LDH,N) On input the initial N-by-N section of H stores the Hessenberg matrix undergoing aggressive early deflation. On output H has been transformed by an orthogonal similarity transformation, perturbed, and the returned to Hessenberg form that (it is to be hoped) has some zero subdiagonal entries. /param>
Parameters
LDH(input) integer Leading dimension of H just as declared in the calling subroutine. N .LE. LDH /param>
Parameters
ILOZ(input) INTEGER /param>
Parameters
IHIZ(input) INTEGER Specify the rows of Z to which transformations must be applied if WANTZ is .TRUE.. 1 .LE. ILOZ .LE. IHIZ .LE. N. /param>
Parameters
Z(input/output) DOUBLE PRECISION array, dimension (LDZ,IHI) IF WANTZ is .TRUE., then on output, the orthogonal similarity transformation mentioned above has been accumulated into Z(ILOZ:IHIZ,ILO:IHI) from the right. If WANTZ is .FALSE., then Z is unreferenced. /param>
Parameters
LDZ(input) integer The leading dimension of Z just as declared in the calling subroutine. 1 .LE. LDZ. /param>
Parameters
NS(output) integer The number of unconverged (ie approximate) eigenvalues returned in SR and SI that may be used as shifts by the calling subroutine. /param>
Parameters
ND(output) integer The number of converged eigenvalues uncovered by this subroutine. /param>
Parameters
SR(output) DOUBLE PRECISION array, dimension KBOT /param>
Parameters
SI(output) DOUBLE PRECISION array, dimension KBOT On output, the real and imaginary parts of approximate eigenvalues that may be used for shifts are stored in SR(KBOT-ND-NS+1) through SR(KBOT-ND) and SI(KBOT-ND-NS+1) through SI(KBOT-ND), respectively. The real and imaginary parts of converged eigenvalues are stored in SR(KBOT-ND+1) through SR(KBOT) and SI(KBOT-ND+1) through SI(KBOT), respectively. /param>
Parameters
V(workspace) DOUBLE PRECISION array, dimension (LDV,NW) An NW-by-NW work array. /param>
Parameters
LDV(input) integer scalar The leading dimension of V just as declared in the calling subroutine. NW .LE. LDV /param>
Parameters
NH(input) integer scalar The number of columns of T. NH.GE.NW. /param>
Parameters
T(workspace) DOUBLE PRECISION array, dimension (LDT,NW) /param>
Parameters
LDT(input) integer The leading dimension of T just as declared in the calling subroutine. NW .LE. LDT /param>
Parameters
NV(input) integer The number of rows of work array WV available for workspace. NV.GE.NW. /param>
Parameters
WV(workspace) DOUBLE PRECISION array, dimension (LDWV,NW) /param>
Parameters
LDWV(input) integer The leading dimension of W just as declared in the calling subroutine. NW .LE. LDV /param>
Parameters
WORK(workspace) DOUBLE PRECISION array, dimension LWORK. On exit, WORK(1) is set to an estimate of the optimal value of LWORK for the given values of N, NW, KTOP and KBOT. /param>
Parameters
LWORK(input) integer The dimension of the work array WORK. LWORK = 2*NW suffices, but greater efficiency may result from larger values of LWORK.
If LWORK = -1, then a workspace query is assumed; DLAQR3 only estimates the optimal workspace size for the given values of N, NW, KTOP and KBOT. The estimate is returned in WORK(1). No error message related to LWORK is issued by XERBLA. Neither H nor Z are accessed. /param>
