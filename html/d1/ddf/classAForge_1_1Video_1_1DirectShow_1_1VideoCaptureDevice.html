Detailed Description

Video source for local video capture device (for example USB webcam).

This video source class captures video data from local video capture device, like USB web camera (or internal), frame grabber, capture board - anything which supports DirectShow interface. For devices which has a shutter button or support external software triggering, the class also allows to do snapshots. Both video size and snapshot size can be configured.

Sample usage:

// enumerate video devices
videoDevices = new FilterInfoCollection( FilterCategory.VideoInputDevice );
// create video source
VideoCaptureDevice videoSource = new VideoCaptureDevice( videoDevices[0].MonikerString );
// set NewFrame event handler
videoSource.NewFrame += new NewFrameEventHandler( video_NewFrame );
// start the video source
videoSource.Start( );
// ...
// signal to stop when you no longer need capturing
videoSource.SignalToStop( );
// ...
private void video_NewFrame( object sender, NewFrameEventArgs eventArgs )
{
// get new frame
Bitmap bitmap = eventArgs.Frame;
// process the frame
}

◆ DisplayCrossbarPropertyPage()

void AForge.Video.DirectShow.VideoCaptureDevice.DisplayCrossbarPropertyPage ( IntPtr  parentWindow)
inline

Display property page of video crossbar (Analog Video Crossbar filter).

Parameters
parentWindowHandle of parent window.

The Analog Video Crossbar filter is modeled after a general switching matrix, with n inputs and m outputs. For example, a video card might have two external connectors: a coaxial connector for TV, and an S-video input. These would be represented as input pins on the filter. The displayed property page allows to configure the crossbar by selecting input of a video card to use.

<note>This method can be invoked only when video source is running (IsRunning is true). Otherwise it generates exception.</note>

Use CheckIfCrossbarAvailable method to check if running video source provides crossbar configuration.

Exceptions
ApplicationExceptionThe video source must be running in order to display crossbar property page.
NotSupportedExceptionCrossbar configuration is not supported by currently running video source.

Property Documentation

◆ CrossbarVideoInput

VideoInput AForge.Video.DirectShow.VideoCaptureDevice.CrossbarVideoInput
getset

Current video input of capture card.

The property specifies video input to use for video devices like capture cards (those which provide crossbar configuration). List of available video inputs can be obtained from AvailableCrossbarVideoInputs property.

To check if the video device supports crossbar configuration, the CheckIfCrossbarAvailable method can be used.

<note>This property can be set as before running video device, as while running it.</note>

By default this property is set to VideoInput.Default, which means video input will not be set when running video device, but currently configured will be used. After video device is started this property will be updated anyway to tell current video input.

Referenced by AForge.Video.DirectShow.VideoCaptureDeviceForm.okButton_Click().

◆ AvailableCrossbarVideoInputs

VideoInput []? AForge.Video.DirectShow.VideoCaptureDevice.AvailableCrossbarVideoInputs
get

Available inputs of the video capture card.

The property provides list of video inputs for devices like video capture cards. Such devices usually provide several video inputs, which can be selected using crossbar. If video device represented by the object of this class supports crossbar, then this property will list all video inputs. However if it is a regular USB camera, for example, which does not provide crossbar configuration, the property will provide zero length array.

Video input to be used can be selected using CrossbarVideoInput. See also DisplayCrossbarPropertyPage method, which provides crossbar configuration dialog.

<note>It is recomended not to call this property immediately after Start method, since device may not start yet and provide its information. It is better to call the property before starting device or a bit after (but not immediately after).</note>

◆ ProvideSnapshots

bool AForge.Video.DirectShow.VideoCaptureDevice.ProvideSnapshots
getset

Specifies if snapshots should be provided or not.

Some USB cameras/devices may have a shutter button, which may result into snapshot if it is pressed. So the property specifies if the video source will try providing snapshots or not - it will check if the camera supports providing still image snapshots. If camera supports snapshots and the property is set to true, then snapshots will be provided through SnapshotFrame event.

Check supported sizes of snapshots using SnapshotCapabilities property and set the desired size using DesiredSnapshotSize property.

<note>The property must be set before running the video source to take effect.</note>

Default value of the property is set to false.

Desired size of captured video frames.

The property sets desired video frame size. However capture device may not always provide video frames of configured size due to the fact that the size is not supported by it.

If the property is set to size (0, 0), then capture device uses its own default video frame size configuration.

Default value of the property is set to (0, 0).

<note>The property should be configured before video source is started to take effect.</note>
